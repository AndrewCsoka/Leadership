# Report writing (draft)

- Subject: if they read nothing else, this should be enough. It should convey the status and what action they need to take. Tailor it to your audience: a CTO needs impact and escalation ("Project is amber - needs more resource"), a project manager needs progress and decisions ("Project is amber - backend delayed, need decision on scope by Wednesday")
- it should answer their question of: what action do i need to take
- use plain language
- avoid saying steps or what youve done
- consistent layout each time
- use RAG colour statuses
- trend is also helpful: amber improving vs amber deteriorating
